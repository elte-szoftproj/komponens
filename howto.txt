Framework
-----
- A kompgame-ejb osztaly szerintem maradjon kozos mindkettoben, folosleges duplikalni, inkabb csak bonyodalmakhoz vezetne...
- A kompgame-web projektet kell lemasolni (ketszer, hogy ketto legyen, az lesz a ket frameworunk. Igy egyszerre lehet ket peldanyban, ket kulon URL-en hasznalni
- A jelenlegi web projekt meg nem mukodik :D Mukodesre kell birni / be kell fejezni (perpillanat login megy, user regisztracio megy, probalok rajonni, hogy kell nev alapjan elerni session beaneket, es utana a jatek is mehet... perpillanat elszallok sajnos egy exceptionnal, pedig onnan mar nem lenne annyira messze, hogy a jatek megjelenitesi resze meglegyen. Jateklista, ilyesmik meg egyaltalan nincs, en azzal nem is akarok foglalkozni)
- Az a lenyeg, hoy valami szep implementaciokat csinaljunk, pl. egy sima htmleset - egy ajaxosat - egy canvasosat - vagy ami eszetekbe jut
- A jatekoknak lesznek statikus assetjei (kepek / css / stb), ezeket a mar most is meglevo web projektbe tesszuk, majd onnan kell oket behivatkozni

Game
-----
- hu.elte.komp.game.GameInterface -t kell implementalni (igazabol az AbstractGame-bol szarmaztassatok!)
- emellett kell egy vagy tobb ScoreCalculator is a jatekhoz, ez megegyezes kerdese, mas is irhat (pl. aisok) ilyen reszkomponenseket, de akkor dolgozzatok ossze
- ez egy olyan osztaly, ami el tudja menteni magat egy Stringkent, lehetoleg rovid formaban, valamint meg tudja jeleniteni magat egy NxN-es tablazatkent
- ha olyan jatekot akartok, aminek nem csak egy tablazat kell, irjatok, es kitalalunk valamit. Elso korben azert probalkoznek egy ilyennel, mert eleg egyszeru interfeszt ad, es igy konnyu implementalni nagyon mashogy mukodo UI-kat ele.
- egyelore probaljatok azt a reszet implementalni a dolognak, ami mar egesz fixen megvan, tehat pl. celszeru egy String <-> Board konverziot csinalni, meghatarozni a valid kattintasok listajat, stb...
- adatbazis mentessel ne foglalkozzatok, meg lesz egy "saveStep" jellegu eljaras az abstractgameben, mert a stat miatt amugy is kelleni fog.
- FONTOS: kattintas != lepes. Pl. a sakkban egy lepes ket kattintasbol all. Az elso kivalasztja a babut, a masodik pedig azt, hogy hova lepunk vele.
